This guide walks through migrating the vault from Obsidian Publish to Quartz 4, a free open-source static site generator built for Obsidian vaults. Quartz gives you a fully interactive graph view (with custom node colors by folder), full-text search, backlinks, wikilink resolution, and mobile support — all hosted for free on GitHub Pages with your own custom domain if you want one.

Prerequisites

  • Node.js v22 or later (check with node -v)
  • npm v10.9.2 or later (comes with Node)
  • Git installed
  • A GitHub account

Step 3 — Configure Quartz

Edit quartz.config.ts in the root of the project:

Key things to configure:

  • pageTitle: What shows in the browser tab and header. “Culture Vault” or whatever you want.
  • baseUrl: If deploying to GitHub Pages at yourusername.github.io/culture-site, set it to that. If using a custom domain like culture.firebirdmag.com, set it to culture.firebirdmag.com.
  • ignorePatterns: Add "META" if you don’t want the “Using this vault” internal notes published. Or leave it out if you do.
  • enablePopovers: Set true — this gives you Wikipedia-style link previews on hover, which is great for a vault like this.

Step 5 — Preview locally

Build and preview your site:

Open http://localhost:8080/ in your browser. You should see your homepage, the full graph with colored nodes, working wikilinks, backlinks, and search. Check both the global graph (click the graph icon) and local graphs on individual pages.

Step 6 — Push to GitHub

Create a new repository on GitHub (e.g. culture-site). Then:

npx quartz sync is Quartz’s built-in command that stages, commits, and pushes to your remote in one step.

Step 7 — Deploy to GitHub Pages

Create the file .github/workflows/deploy.yml in your repository:

Then on GitHub, go to your repo → Settings → Pages → under “Source”, select “GitHub Actions”.

Push the workflow file, and GitHub will automatically build and deploy your site every time you push changes.

Step 8 — Custom domain (optional)

If you want to host at something like culture.firebirdmag.com:

  1. In your DNS provider, add a CNAME record pointing culture.firebirdmag.com to yourusername.github.io
  2. On GitHub, go to repo → Settings → Pages → Custom domain → enter culture.firebirdmag.com
  3. In quartz.config.ts, set baseUrl: "culture.firebirdmag.com"
  4. Push the changes

Ongoing workflow

Once this is set up, your daily workflow is:

  1. Write and edit notes in Obsidian as usual
  2. When you want to publish changes, run npx quartz sync from the quartz folder (or just git add . && git commit -m "update" && git push if you prefer standard git)
  3. GitHub Actions builds and deploys automatically — takes about 1–2 minutes

If you used the symlink approach in Step 2, your Obsidian edits are already in the quartz content folder. If you used the copy approach, you’ll need to re-copy changed files before syncing.
